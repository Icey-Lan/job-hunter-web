    • 需求来自BOSS直聘分析
    • 数据分析
    岗位职责：
    1、【数据资产建设】
    ○ 抽象业务需求，主导各数据域建设资产，进行横向、体系化的数据规划。并建立体系化的数据资产标准（统一模型、质量SLA），缩短业务数据获取与准备周期，使得能够快速响应数据分析、报表开发、业务决策、数据应用等场景业务需求。
    2、【机制驱动与规模化】
    ○ 推动数据资产建设流程由依赖“人治”转变为依赖“机制”。建立一套体系化、流程化的资产建设、分级、评审与维护规范，杜绝单点需求交付、重复建设，以流程保障平台的可持续发展和规模化效率。
    3、【可信资产门户】
    ○ 打造企业级官方权威数据资产字典，正式推出。作为全公司统一的数据目录和知识库，它将帮助大家低门槛、快速地发现、理解、评估所需数据，
    4、【数据资产管理工具】：
    ○ 面对数据供给者团队，建立一套简易高效率的资产维护工具，帮助数据供给者高效、高质量地维护数据资产。并作为平台方推行资产分级管理和数据运营机制。
    5、【智能化落地】
    ○ 面向数据使用者，满足数据发现 、数据检索能力，并向AI 智能化 Agent 进化。 实现 分析师的智能分身Agent， 可落地接管数据查找与咨询类场景， 数据对外进行全面推广应用。实现分析师的智能分身Agent落地，并为Agent的使用量和准确度负责
    任职要求：
    该岗位需要一位具有业务视角、体系化思维、数据背景，且具备极强的跨部门沟通和推动能力，并可协同多个相关方建立起可落地机制的 产品经理。
    【数据管理模式升级与落地】 彻底改变当前数据资产“小作坊”建设模式，建立起有效的、可依赖的机制保障。
    【核心资产平台化】 夯实数据资产平台能力，并推动使数据供给团队将资产主动沉淀至平台中。正式建立并推出官方权威的数据资产字典/门户。
    【数据消费体验与效率革命】 实现分析师智能分身 Agent 落地，有效减轻数据支持团队对数据查找与咨询的重复性支持工作。
    ● 核心定位： 企业级数据资产管理体系，负责将数据管理从“人治”的“小作坊”模式升级为**“机制”保障的规模化模式**。
    ● 人才特质： 有业务视角 + 体系化思维 + 习惯建立机制流程解决问题 + 极强的跨部门沟通和推动能力。
    ● 经验： 5年+的数据产品数据建设经验
